# Build Your First Motia App

**Get up and running with Motia in just a few minutes!** This guide shows you how to create a Motia app that connects JavaScript, TypeScript, and Python as a steps.

## What You'll Build

A simple data processing Motia app:
- **JavaScript** API endpoint receives data
- **TypeScript** bridges and notifies
- **Python** processes the data

All connected automatically with zero configuration.

## Step 1: Create Your Motia App

```bash
# Create a new Motia app
npx motia@latest create -n my-app
```

## Step 2: Add Your Logic Steps

### JavaScript API Endpoint (App Starter)
// File: 01-starter.step.js
```javascript
// Basic app starter - JavaScript API endpoint
export const config = {
  type: 'api',
  name: 'appStarter',
  description: 'Start the basic multi-language app',

  method: 'POST',
  path: '/start-app',

  emits: ['app.started'],
  flows: ['data-processing']
}

export const handler = async (req, { logger, emit, traceId }) => {
  logger.info('üöÄ Starting basic app', { body: req.body, traceId })
  
  const appData = {
    id: Date.now(),
    input: req.body,
    started_at: new Date().toISOString(),
    traceId: traceId
  }
  
  // Emit to start the app
  await emit({
    topic: 'app.started',
    data: appData
  })
  
  logger.info('app initiated successfully', { appId: appData.id })
  
  return {
    status: 200,
    body: {
      message: 'Basic app started successfully',
      appId: appData.id,
      traceId: traceId
    }
  }
} 
```

### TypeScript Bridge Step
// File: 02-bridge.step.ts
```typescript
// Bridge step to connect app starter to Python processing
export const config = {
  type: 'event',
  name: 'appBridge',
  description: 'Bridge between app start and Python processing',
  
  subscribes: ['app.started'],
  emits: ['data.processed'],
  
  flows: ['data-processing']
}

export const handler = async (input: any, { logger, emit }: any) => {
  logger.info('üåâ Processing app data and sending to Python', { appId: input.id })
  
  // Process data for Python step
  const processedResult = {
    original_id: input.id,
    processed_at: input.started_at,
    result: `Processed: ${JSON.stringify(input.input)}`,
    confidence: 0.95,
    model_version: 'v2.1-ts'
  }
  
  // Send to Python step for async processing
  await emit({
    topic: 'data.processed',
    data: processedResult
  })
  
  logger.info('Data sent to Python step for processing', { dataId: processedResult.original_id })
  
  return processedResult
} 
```

### Python Data Processor
// File: process-data.step.py
```python
# process-data.step.py
config = {
    'type': 'event',
    'name': 'ProcessDataPython',
    'description': 'Process incoming data and emit python.done',
    'subscribes': ['data.processed'],
    'emits': ['python.done'],
    'flows': ['data-processing']
}

async def handler(input_data, context):
    print(f"üêç Received: {input_data['original_id']}")

    # 1. Do your processing
    python_result = {
        'id': input_data['original_id'],
        'python_message': f"Python processed: {input_data['result']}",
        'processed_by': 'python-step',
        'timestamp': input_data['processed_at']
    }

    print(f"üêç Processing complete, emitting python.done event")

    # 2. Emit with topic and data in dictionary format
    await context.emit({"topic": "python.done", "data": python_result})

    print(f"üêç Event emitted successfully")

    # 3. Return the payload so Motia passes it along automatically
    return python_result
```

### TypeScript Notification Step
// File: notify.step.ts
```typescript
import { EventConfig, Handlers } from 'motia'

interface PythonResult {
  id: number
  python_message: string
  processed_by: string
  timestamp: string
}

export const config = {
  type: 'event',
  name: 'NotificationHandler',
  description: 'Send notifications after Python processing',
  
  subscribes: ['python.done'],
  emits: ['notification.sent'],
  
  flows: ['data-processing']
}

export const handler = async (input: PythonResult, { logger, emit }: any) => {
  logger.info('üìß Sending notifications after Python processing:', { id: input.id })
  
  // Simulate sending notifications (email, slack, etc.)
  const notification = {
    id: input.id,
    message: `Notification: ${input.python_message}`,
    processed_by: input.processed_by,
    sent_at: new Date().toISOString()
  }
  
  // Trigger final step
  await emit({
    topic: 'notification.sent',
    data: notification
  })
  
  logger.info('Notification sent successfully', notification)
  
  return notification
}
```

### JavaScript Finalizer Step
// File: 04-final.step.js
```javascript
// Final step to complete the app - JavaScript
export const config = {
  type: 'event',
  name: 'appFinalizer',
  description: 'Complete the basic app and log final results',
  
  subscribes: ['notification.sent'],
  emits: ['app.completed'],
  
  flows: ['data-processing']
}

export const handler = async (input, { logger, emit }) => {
  logger.info('üèÅ Finalizing app', { 
    notificationId: input.id,
    message: input.message 
  })
  
  // Create final app summary
  const summary = {
    appId: input.id,
    status: 'completed',
    completed_at: new Date().toISOString(),
    steps_executed: [
      'appStarter (JavaScript)',
      'appBridge (TypeScript)', 
      'ProcessDataPython (Python)',
      'NotificationHandler (TypeScript)',
      'appFinalizer (JavaScript)'
    ],
    result: input.message,
    confidence: input.confidence
  }
  
  // Emit completion event
  await emit({
    topic: 'app.completed',
    data: summary
  })
  
  logger.info('‚úÖ Basic app completed successfully', summary)
  
  return summary
} 
```

## What Happens Next?

![motia-build-your-app](/docs-images/motia-build-your-app-2.gif)

Watch in the **Workbench** (`http://localhost:3000`) as your data flows through:

1. **JavaScript** API receives the request
2. **Python** processes it (with access to numpy, pandas, torch, etc.)
3. **TypeScript** handles notifications (with full type safety)

All languages working together in one unified system with:
- ‚úÖ **Automatic observability** - see every step in real-time
- ‚úÖ **Built-in error handling** - retry logic included
- ‚úÖ **Shared state** - pass data between languages effortlessly
- ‚úÖ **Hot reload** - edit any file and see changes instantly
