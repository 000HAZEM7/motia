## The Problem Motia Solves

Building production-grade AI agents is tough. You're usually forced to choose:

- **No-code/Low-code tools:** Quick to start, but can't handle real-world complexity at scale.
- **Fully-agentic Frameworks:** Great for creative tasks but difficult to manage and control consistent outputs.
- **ML-focused Frameworks:** Designed for model training and optimization, not for orchestrating complex business workflows.
- **Custom code:** Quickly becomes unmanageable as agents grow in complexity and span across teams.

Motia gives you full, code-first control of your agents and automations with the simplicity of a visual interface, letting you focus on what truly matters: your business logic, not your infrastructure.

## Quick Start

Install Motia using your preferred package manager:

```bash
npm install motia
# or
yarn add motia
# or
pnpm add motia
```

Create your first agent:

<CodeDisplay
  language="typescript"
  code={`import { OpenAI } from 'openai';
import { z } from 'zod';
import type { EventConfig, StepHandler } from 'motia';

// Define a simple step
export const config: EventConfig = {
  type: 'event',
  name: 'sentiment-analyzer',
  subscribes: ['text.received'],
  emits: ['sentiment.analyzed'],
  input: z.object({ text: z.string() }),
};

export const handler: StepHandler<typeof config> = async (inputData, context) => {
  const { text } = inputData;
  const { emit, logger } = context;
  
  // Process the text (in a real app, you might use an LLM here)
  const sentiment = text.includes('love') || text.includes('great') 
    ? 'positive' 
    : 'neutral';
  
  logger.info('Analyzed sentiment', { sentiment });
  
  // Emit the result
  emit({
    type: 'sentiment.analyzed',
    data: { text, sentiment }
  });
  
  return { success: true };
};`}
/>

## Motia Workbench

The Motia Workbench is your browser-based development environment that makes building and debugging agents intuitive:

<div className="my-8">
  <img 
    src="/images/gmail-example.png" 
    alt="Motia Workbench Interface showing a workflow visualization" 
    className="rounded-lg border border-gray-200 shadow-md w-full"
  />
</div>

- **Interactive Flow Visualization:** See your steps connected in a dynamic, visual graph
- **Real-time Testing:** Trigger API endpoints and emit events directly from the UI
- **Live Log Streaming:** Monitor logs in real-time within the Workbench
- **Step Customization:** Create custom UI components for your steps
